# Moleben

Moleben — backend-сервис для диалога с AI-собеседником в формате «исповеди». Сервис помогает пользователю осмыслить поступок, оценивает выраженное в текущем сообщении раскаяние и ведёт общий счётчик сессии.

> Проект не заменяет психологическую, медицинскую, юридическую или религиозную помощь специалиста.

## Как это работает

1. Клиент создаёт сессию. Начальное значение счётчика (`tally`) равно `50`.
2. Пользователь отправляет сообщение в сессию.
3. Backend передаёт системный промпт и сообщение в LLM через OpenAI-совместимый API.
4. Модель возвращает JSON с ответом, оценкой раскаяния, обнаруженными сигналами и рекомендуемым следующим шагом.
5. Сервис обновляет счётчик по формуле `tally += score - 50`, ограничивая результат диапазоном `0..100`.
6. При достижении `100` сессия завершается результатом «вы прощены», при достижении `0` — «вы не прощены».

Если модель обнаруживает признаки самоповреждения, суицида или насилия, она выставляет `risk_flag: true`. В этом случае оценка не влияет на счётчик.

Каждое сообщение сейчас оценивается независимо: история сессии сохраняется в PostgreSQL, но не передаётся модели как контекст.

## Возможности

- создание и завершение чат-сессий;
- интеграция с OpenRouter и DeepSeek;
- структурированный JSON-ответ AI-собеседника;
- оценка текущего раскаяния по шкале от 0 до 100;
- фиксация факторов, повлиявших на оценку;
- обработка сообщений с признаками непосредственного риска;
- хранение сессий и истории сообщений в PostgreSQL;
- автоматический запуск миграций при старте;
- health-check endpoint;
- Swagger-спецификация API.

## Быстрый запуск

### 1. Запустите PostgreSQL

```bash
docker compose -f docker/docker-compose.yml up -d db
```

По умолчанию база будет доступна по адресу `localhost:5432`:

```text
postgres://moleben:moleben@localhost:5432/moleben?sslmode=disable
```

### 2. Настройте окружение

Для OpenRouter:

```bash
export PROVIDER=openrouter
export API_KEY=your_openrouter_api_key
export MODEL=nvidia/nemotron-nano-9b-v2:free
```

Для DeepSeek:

```bash
export PROVIDER=deepseek
export API_KEY=your_deepseek_api_key
export MODEL=deepseek-chat
```

### 3. Запустите API

```bash
go run .
```

При запуске приложение автоматически применит миграции из каталога `migrations`. API будет доступен по адресу `http://localhost:8080`.

Проверка:

```bash
curl http://localhost:8080/health
```

Ожидаемый ответ:

```text
ok
```

## Конфигурация

Настройки задаются переменными окружения.

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `PROVIDER` | `openrouter` | Провайдер LLM: `openrouter` или `deepseek` |
| `BASE_URL` | зависит от провайдера | Базовый URL OpenAI-совместимого API |
| `MODEL` | зависит от провайдера | Идентификатор модели |
| `API_KEY` | в коде присутствует небезопасный fallback | API-ключ провайдера; обязательно переопределите через окружение |
| `DB_DSN` | `postgres://moleben:moleben@localhost:5432/moleben?sslmode=disable` | Строка подключения к PostgreSQL |
| `MIGRATIONS_DIR` | `./migrations` | Каталог миграций Goose |
| `HTTP_ADDR` | `:8080` | Адрес HTTP-сервера |
| `PROMPT_PATH` | `prompt` | Путь к системному промпту |
| `APP_REFERER` | `http://localhost:3000` | Referer для OpenRouter |
| `APP_TITLE` | `Moleben` | Название приложения для OpenRouter |

## API

### Создать сессию

```http
POST /api/sessions
```

Пример:

```bash
curl -X POST http://localhost:8080/api/sessions
```

Ответ:

```json
{
  "id": "3d993754-9468-4d57-9213-ace9276166f4",
  "tally": 50,
  "status": "active"
}
```

### Отправить сообщение

```http
POST /api/sessions/{id}/messages
Content-Type: application/json
```

```bash
curl -X POST http://localhost:8080/api/sessions/SESSION_ID/messages \
  -H 'Content-Type: application/json' \
  -d '{"text":"Я поступил неправильно и хочу исправить причинённый вред"}'
```

Пример ответа:

```json
{
  "assistant": {
    "reply": "Признание ответственности — важный шаг. Что вы готовы сделать для исправления ситуации?",
    "repentance": {
      "score": 72,
      "trend": "up",
      "signals": [
        {
          "factor": "ответственность",
          "direction": "+",
          "evidence": "Пользователь признаёт, что поступил неправильно"
        }
      ],
      "confidence": "medium"
    },
    "next_step": "Сформулировать один конкретный шаг для исправления вреда",
    "risk_flag": false
  },
  "tally": 72,
  "done": false,
  "outcome": null
}
```

### Получить состояние сессии

```http
GET /api/sessions/{id}/
```

```bash
curl http://localhost:8080/api/sessions/SESSION_ID/
```

### Получить историю сообщений

```http
GET /api/sessions/{id}/messages
```

```bash
curl http://localhost:8080/api/sessions/SESSION_ID/messages
```

Метод возвращает первые 200 сообщений в хронологическом порядке.

Полная спецификация находится в [`docs/swagger.yaml`](docs/swagger.yaml). Swagger UI в HTTP-сервер пока не подключён.

## Docker

Образ приложения можно собрать командой:

```bash
docker build -f docker/Dockerfile -t moleben .
```

Для полного запуска контейнеру необходимо передать как минимум `API_KEY` и корректный `DB_DSN`. Текущий `docker-compose.yml` поднимает PostgreSQL и API, но не пробрасывает настройки LLM из окружения; перед использованием полного Compose-стека добавьте их в секцию `api.environment`.

## Ограничения текущей версии

- отсутствуют авторизация и разделение данных по пользователям;
- история диалога не передаётся LLM;
- запрос пользователя сохраняется до успешного ответа LLM, поэтому при ошибке провайдера в истории останется сообщение без ответа;
- обновление счётчика и запись сообщений выполняются без общей транзакции;
- Swagger UI не подключён;
- лимиты запросов и защита от злоупотреблений не реализованы;
- проект содержит только backend, пользовательский интерфейс отсутствует.

## Безопасность

- Не храните API-ключи в исходном коде и не коммитьте их в Git.
- Если ключ когда-либо попадал в репозиторий, отзовите его у провайдера и выпустите новый.
- Перед публичным развёртыванием добавьте авторизацию, rate limiting и ограничения размера входящих сообщений.
- Не используйте текущую оценку модели как профессиональное заключение или основание для решений с серьёзными последствиями.
